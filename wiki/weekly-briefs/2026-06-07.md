# Trends in the industry

- **Agent runtime infrastructure is becoming a peer-reviewed subfield.** Two independent papers this week formalise what was previously ad-hoc middleware: Agent libOS (2606.03895) applies library-OS and object-capability concepts (process identity, capability tables, namespace-local memory, JIT tool generation) to long-running agents; the Organizational Control Layer (2606.04306) does the same for governance, intercepting agent proposals at the execution boundary before they affect the environment. The convergence point: harness engineering is no longer just practitioner lore — it has a formal systems-research literature now.
- **Memory security is a structurally different problem from prompt injection.** MPBench (2606.04329, ICML 2026) demonstrates that all existing prompt injection defenses fail against memory poisoning — not because they are incomplete, but because they operate at the input boundary while weak-signal poisoning attacks only reveal adversarial intent when evaluated against what the agent is *authorised to store*. One successful write poisons every future session. This is the first empirical proof that the threat model for persistent-memory agents is qualitatively different from the stateless case.
- **The first cross-framework ADK comparison lands, and the supply-chain finding is the headline.** ADK Arena (2606.05548) uses an LLM-as-Developer protocol to evaluate 51 Python agent frameworks without writing a single benchmark by hand. The framework quality finding (Opus-authored agents 2× GPT-Turbo-authored) matters, but the structural finding matters more: 74% of inter-ADK dependencies are undeclared code-only imports, and pydantic is a single point of failure for 75% of the ecosystem. Framework selection is not a neutral choice — it is also a supply-chain decision.
- **SWE-bench Verified is approaching saturation (93.9%), shifting attention to harder evals.** The contamination gap between Verified (93.9%, Claude Mythos Preview) and Pro (77.8%) has become a debate in its own right. OSWorld shows a 2× spread between the 82% leader and major-lab baselines. The GAIA scaffold gap (74.6% bare vs ~92% scaffolded) now dominates the evaluation-methodology discussion: at the frontier, harness quality may be the primary performance driver, not model quality. The field is hunting for benchmarks that can't be gamed by scaffold engineering.
- **Agent memory formalisation is catching up to production.** GEM (arXiv 2605.26252) is the most rigorous attempt yet to specify what "correct" long-term agent memory means: correctness is a property of the *state trajectory*, not of individual records. Four state-level operators, six correctness conditions, three impossibility observations. The MemState prototype on Kuzu exposes three native-engine gaps — concrete engineering targets for anyone building production memory systems. This is complementary to MPBench's finding: GEM says what correct memory *is*; MPBench says what happens when the write channel is unprotected.
