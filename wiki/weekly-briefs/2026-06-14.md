# Trends in the industry

- **The benchmark ceiling is hardening and disaggregating simultaneously.** ALE's Last-Exam tier (38 professional-workflow tasks) scores 0% full pass rate across every tested frontier agent — the same week that SWE-bench Verified reportedly crossed 95%. This is not a contradiction: ALE explicitly shows that Terminal-Bench performance (82% for Codex + GPT-5.5) is non-predictive of ALE-CLI performance (23.3% for the same agent). The field is not reaching saturation — it was measuring the wrong things. SWE-Explore adds another layer: agents locate the right files but fail to cover the specific line spans they need (Rec_ℓ ~0.14–0.19 for all frontier models), and this line-level gap is the strongest predictor of repair success (r=0.950). The emerging picture: coding agents are competent in narrow slices, and evaluation was previously missing the binding bottleneck.

- **Coding agent adoption has crossed an inflection point; pervasive use is now the mode.** 71.83% of new GitHub projects (created post-August 2025) show conservative coding agent adoption — 2.7× higher than older projects, with "Pervasive" (>20% AI commits) as the most common bin at 41.2%. Claude Code leads at ~3× GitHub Copilot's volume. This empirically grounds the Willison PMF signal from May: adoption is not a trend, it's the new baseline for projects started today. Enterprise orgs (Microsoft, Amazon, Nvidia, Google) are now at parity with the community average — the enterprise lag has closed.

- **Agent memory is a systems problem, not an accuracy problem.** The first cross-paradigm energy characterization (10 systems, 4 paradigms) shows BM25 wins on J/correct-answer — 47× cheaper per correct answer than Letta, while achieving the highest macro-averaged accuracy (55.8%). LLM-mediated systems (Mem0, A-Mem, Letta) have construction costs that dominate their total lifecycle energy, scale super-linearly with store size, and have tail latency up to 5.9× the median. The takeaway for practitioners: the accuracy gap between BM25 and neural systems is narrower than the cost gap is wide. This is the third independent benchmark showing BM25 competitive on knowledge tasks (after GroupMemBench and EvoMemBench).

- **Environment engineering is displacing workflow prescription as the primary agent design lever.** EurekAgent (Tsinghua/Zhipu AI) achieves new SOTA on five scientific tasks — training-free, using off-the-shelf CLI agents — by engineering four dimensions of the environment (permissions, artifacts, budgets, human-in-the-loop) rather than specifying what the agent should do. Open-source GLM-5.1 beats baselines using Claude Opus 4.6 and Gemini-2.5-Pro. The framing: as CLI agents mature, the bottleneck shifts from "how do I tell the agent what to do" to "how do I structure the environment around the agent." This is a clean empirical rebuttal to workflow-prescription approaches.

- **The vendor/standardized-harness divergence on benchmarks is now a first-class problem.** (collect-but-confirm, not captured) SWE-bench Verified reportedly reached 95% while SWE-bench Pro standardized harness shows GPT-5.4 at 59.1% for a comparable system. OSWorld's leader is now a commercial product (Coasty, 82%, above human baseline) rather than a lab model. The pattern: vendor-reported benchmark scores are converging toward saturation while standardized harness numbers remain well below that — a systematic gap that needs a methodology paper.

- **ALE establishes that model choice matters ~3× more than harness choice for professional tasks.** Foundation model choice accounts for roughly 3× the performance spread that harness choice does among well-engineered systems. Domain knowledge failures (~75% of ALE failures) are the binding constraint, not execution capability failures. This reframes the harness vs model scaling debate: harness evolution matters, but it can't compensate for domain knowledge gaps that are fundamental to the model.
