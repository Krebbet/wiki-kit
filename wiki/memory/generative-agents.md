# Generative Agents: Interactive Simulacra of Human Behavior

Stanford + Google paper (Joon Sung Park, Joseph C. O'Brien, Carrie J. Cai, Meredith Ringel Morris, Percy Liang, Michael S. Bernstein; arXiv 2304.03442; UIST 2023) that demonstrates 25 LLM agents living in the **Smallville** sandbox — a Sims-like 2D world — and exhibiting believable, character-driven behaviour over multi-day simulations. The contribution that has propagated through nearly every subsequent memory architecture is two-fold: (1) the **memory stream** as a unified first-class container for observations, reflections, and plans, and (2) the **retrieval scoring formula** that selects what to load into a finite context window from an unbounded experience log. The formula — `score = α_recency · recency + α_importance · importance + α_relevance · relevance` with min-max normalisation — became standard. Mem0 ([[mem0]]) and many others adopt variants. Implementation model in the original paper: gpt-3.5-turbo.

## The memory stream

A list of memory objects, each carrying:
- A natural-language description of the event.
- A creation timestamp.
- A most-recent-access timestamp.

Three entry types share the stream and the same retrieval pipeline:
- **Observations** — events directly perceived: the agent's own actions or things it observes happening to others/objects (e.g., *"Isabella Rodriguez is setting out the pastries"*; *"The refrigerator is empty"*).
- **Reflections** — higher-level abstract thoughts generated by the agent.
- **Plans** — future action sequences with location, start time, duration.

The stream grows continuously; retrieval surfaces a relevant subset to pass to the LLM at each reasoning step.

## Retrieval scoring

**`score = α_recency · recency + α_importance · importance + α_relevance · relevance`**

In the paper's implementation **all α weights are 1** (equal weighting). All three raw scores are normalised to [0, 1] via **min-max scaling**. Top-scoring memories that fit within the context window are included.

- **Recency** — exponential decay over sandbox hours since last access. **Decay factor = 0.995.** Recent events score near 1; old, unaccessed memories decay toward 0.
- **Importance** — LLM-assigned integer on a **1–10 scale** ("poignancy"), generated at memory creation time. Prompt: *"Rate how poignant the memory is, where 1 = purely mundane (brushing teeth) and 10 = extremely poignant (breakup, college acceptance)."* Examples: "cleaning up the room" → 2; "asking your crush out on a date" → 8.
- **Relevance** — cosine similarity between the embedding of the memory's text description and the embedding of a *query memory* representing the current situation. Embeddings generated by the LLM; similarity computed at query time.

## Reflection mechanism

**Trigger** — generated when the sum of importance scores for the most recently perceived events exceeds **150**. In practice agents reflect roughly two or three times per day.

**Process**:
1. LLM is given the **100 most recent memory records** and prompted: *"Given only the information above, what are 3 most salient high-level questions we can answer about the subjects in the statements?"*
2. Generated questions become retrieval queries to fetch relevant memories (including prior reflections).
3. LLM is prompted to extract **5 high-level insights** from retrieved records, citing the specific memory indices that serve as evidence.
4. Resulting statements (e.g., *"Klaus Mueller is dedicated to his research on gentrification (because of 1, 2, 8, 15)"*) are stored as reflections in the memory stream with pointers to evidence.

Reflections can reference other reflections, producing a **tree structure**: leaf nodes are raw observations; higher nodes are increasingly abstract inferences.

## Planning

Hierarchical, top-down:
1. LLM generates a **coarse daily plan** in 5–8 chunks (e.g., *"wake up at 8am… work on music composition 1–5pm… dinner at 5:30pm"*).
2. Recursively decomposed into hour-long chunks, then into 5–15 minute actions with specific locations.
3. Plans are stored in the memory stream and participate in retrieval like any other memory.
4. At each time step the agent evaluates whether to continue or react; if a new observation warrants a change (e.g., seeing a friend taking a walk), the plan is regenerated from that point forward.

## The 25-agent Smallville simulation — emergent behaviours

Over two game days with no user intervention after seeding:
- **Sam's mayoral candidacy** spread from 1 agent (4%) to **8 agents (32%)**.
- **Isabella's Valentine's Day party** spread from 1 agent (4%) to **13 agents (52%)**.
- Network density of mutual agent knowledge increased from 0.167 to **0.74**.
- **5 of 12 invited agents** showed up at the party at the correct time and location.

Direct quote on the seeding-to-emergent-behaviour chain: *"starting with only a single user-specified notion that one agent wants to throw a Valentine's Day party, the agents autonomously spread invitations to the party over the next two days, make new acquaintances, ask each other out on dates to the party, and coordinate to show up for the party together at the right time."*

The simulation also demonstrated relationship memory (Sam later asking Latoya about her photography project by name) and information diffusion cascades.

## Evaluation and ablations

**Controlled "interview" eval** — 100 Prolific participants watched a replay of one agent's life and ranked responses across 5 conditions on **believability** for 5 question categories (self-knowledge, memory, plans, reactions, reflections). Ratings via TrueSkill; Kruskal-Wallis + Dunn post-hoc + Holm-Bonferroni.

| Condition | TrueSkill μ |
|---|---|
| Full architecture | **29.89** |
| No reflection (observations + plans) | 26.88 |
| No reflection, no planning (observations only) | 25.64 |
| Human crowdworker | 22.95 |
| No observation, no reflection, no planning (baseline LLM) | 21.21 |

Effect size between no-memory baseline and full architecture: **Cohen's d = 8.16** (eight standard deviations). All pairwise differences significant at p < 0.001 except crowdworker vs full-ablation baseline.

Per-component findings:
- Without reflection: agents handle factual recall but fail synthesis tasks (e.g., Maria couldn't say what to get Wolfgang for his birthday despite many interactions).
- Without planning: less coherent long-horizon behaviour (eating lunch three times in a row).

## Self-documented limitations

The authors flag explicitly:
- **Memory retrieval failures** — agents sometimes fail to surface relevant memories even when they exist (Rajiv claiming not to follow an election he had heard about).
- **Hallucinated embellishments** — rarely fabricate events wholesale, but add unsupported details (Isabella claiming Sam would *"make an announcement tomorrow"*; Yuriko attributing *Wealth of Nations* to a neighbour named Adam Smith by leveraging world knowledge in the LLM).
- **Incomplete retrieval** — Tom knew what he was supposed to discuss at Isabella's party but couldn't recall that the party existed.
- **Instruction-tuning artifacts** — agents speak overly formally, are excessively cooperative; Isabella accepted ideas contradicting her personality, and her stated interests drifted toward whatever others suggested.
- **Location misclassification** — growing memory caused agents to choose atypical action locations (going to the bar for lunch, treating shared dorm bathrooms as multi-occupancy).
- **Memory hacking / robustness** — vulnerable to crafted conversations implanting false memories.
- **LLM bias inheritance** — biases in the underlying model propagate to agent behaviour.
- **Cost and speed** — simulating 25 agents for two days cost *"thousands of dollars in token credits"* and took multiple days.

## Why it matters in the taxonomy

[[reflexion]] (Shinn et al., 2023) introduced reflective verbal feedback as episodic memory but operated on task trajectories, not a persistent biographical memory. Generative Agents added two pieces that became canonical infrastructure:

1. **The memory stream model** — a continuously growing, timestamped, natural-language record where observations, reflections, and plans are unified first-class objects.
2. **The recency × importance × relevance retrieval formula** — solves the concrete problem of selecting a finite-context-window subset from an unbounded experience log. The three-axis decomposition is principled enough to be reweighted, replaced with learned scorers, or extended.

[[mem0]] (2025) is the production library that operationalises a similar retrieval discipline (with conflict resolution and a graph variant) for agent applications. [[memory-architectures]] places this paper as one of the two most-cited foundational references for *reflective and self-improving memory*, alongside Reflexion.
